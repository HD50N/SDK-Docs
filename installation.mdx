HumanBehavior SDK can be installed in several ways depending on your project setup and preferences.

## Package Managers

### NPM

```bash
npm install humanbehavior-js
```

### Yarn

```bash
yarn add humanbehavior-js
```

### PNPM

```bash
pnpm add humanbehavior-js
```

## CDN

For quick prototyping or when you can't use a package manager:

```html
<script src="https://unpkg.com/humanbehavior-js@latest/dist/index.min.js"></script>
```

<Note>
  The CDN version exposes `HumanBehaviorTracker` globally on the `window` object.
</Note>

## Local File

If you have the SDK files locally (e.g., after building from source):

```html
<script src="./dist/index.min.js"></script>
```

<Note>
  Make sure the path to your `dist/index.min.js` file is correct relative to your HTML file.
</Note>

## Module Types

The SDK supports multiple module formats:

### ES Modules (ESM)

```javascript
import { HumanBehaviorTracker } from 'humanbehavior-js';
```

### CommonJS (CJS)

```javascript
const { HumanBehaviorTracker } = require('humanbehavior-js');
```

### TypeScript

The SDK includes TypeScript definitions:

```typescript
import { HumanBehaviorTracker } from 'humanbehavior-js';

const tracker: HumanBehaviorTracker = HumanBehaviorTracker.init('your-api-key');
```

### Next.js

The SDK is fully compatible with Next.js and server-side rendering:

```javascript
// pages/_app.js
import { HumanBehaviorProvider } from 'humanbehavior-js/react';

export default function App({ Component, pageProps }) {
  return (
    <HumanBehaviorProvider apiKey={process.env.NEXT_PUBLIC_HUMANBEHAVIOR_API_KEY}>
      <Component {...pageProps} />
    </HumanBehaviorProvider>
  );
}
```

### Vue.js

Use the standard JavaScript SDK with Vue:

```javascript
// main.js
import { createApp } from 'vue';
import { HumanBehaviorTracker } from 'humanbehavior-js';
import App from './App.vue';

const app = createApp(App);

// Initialize tracker
const tracker = HumanBehaviorTracker.init('your-api-key');

// Make tracker available globally
app.config.globalProperties.$tracker = tracker;

app.mount('#app');
```

### Raw HTML

For static HTML pages or simple websites, use the CDN version:

```html
<!DOCTYPE html>
<html>
<head>
    <title>My Website</title>
    <!-- Add the HumanBehavior SDK from CDN -->
    <script src="https://unpkg.com/humanbehavior-js@latest/dist/index.min.js"></script>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <button id="contact-btn">Contact Us</button>
    
    <script>
        // Initialize tracker
        const tracker = HumanBehaviorTracker.init('your-api-key', {
            redactFields: ['input[type="password"]', '#email-field']
        });
        
        // Track button clicks
        document.getElementById('contact-btn').addEventListener('click', async () => {
            await tracker.customEvent('contact_button_clicked', {
                buttonId: 'contact-btn',
                page: window.location.pathname
            });
        });
        
        // Track form submissions
        document.querySelectorAll('form').forEach(form => {
            form.addEventListener('submit', async (e) => {
                await tracker.customEvent('form_submitted', {
                    formId: form.id || 'unnamed-form',
                    action: form.action
                });
            });
        });
    </script>
</body>
</html>
```

<Note>
  The script tag must be placed in the `<head>` section before any JavaScript that uses the SDK. This ensures the `HumanBehaviorTracker` class is available globally.
</Note>

## Environment Variables

For security, store your API key in environment variables:

### Development (.env)

```bash
HUMANBEHAVIOR_API_KEY=your-api-key-here
NEXT_PUBLIC_HUMANBEHAVIOR_API_KEY=your-api-key-here
```

### Production

```bash
# Set environment variables in your hosting platform
HUMANBEHAVIOR_API_KEY=your-production-api-key
NEXT_PUBLIC_HUMANBEHAVIOR_API_KEY=your-production-api-key
```

## Initialization Options

The SDK supports various initialization options:

```javascript
const tracker = HumanBehaviorTracker.init('your-api-key', {
  // Fields to redact (CSS selectors)
  redactFields: [
    'input[type="password"]',
    '#email-field',
    '.sensitive-data'
  ],
  
  // Automatic tracking (enabled by default)
  enableAutomaticTracking: true,
  
  // Fine-tune automatic tracking behavior
  automaticTrackingOptions: {
    trackButtons: true,     // Track button clicks
    trackLinks: true,       // Track link clicks  
    trackForms: true,       // Track form submissions
    includeText: true,      // Include button/link text content
    includeClasses: false   // Include CSS classes
  }
});
```

### Automatic Tracking

The SDK can automatically track user interactions without any manual code:

#### Button Tracking
Automatically tracks all button clicks with properties:
- `buttonId`: Button's ID attribute
- `buttonType`: Button type (submit, button, reset)
- `buttonText`: Button text content (if enabled)
- `buttonClass`: CSS classes (if enabled)
- `page`: Current page path
- `timestamp`: Click timestamp

#### Link Tracking  
Automatically tracks all link clicks with properties:
- `linkUrl`: Link destination URL
- `linkId`: Link's ID attribute
- `linkTarget`: Link target attribute (_blank, _self, etc.)
- `linkText`: Link text content (if enabled)
- `linkClass`: CSS classes (if enabled)
- `page`: Current page path
- `timestamp`: Click timestamp

#### Form Tracking
Automatically tracks all form submissions with properties:
- `formId`: Form's ID attribute
- `formAction`: Form action URL
- `formMethod`: Form method (get, post, etc.)
- `fields`: Array of form field names
- `formClass`: CSS classes (if enabled)
- `page`: Current page path
- `timestamp`: Submission timestamp

#### Disabling Automatic Tracking

```javascript
// Disable all automatic tracking
const tracker = HumanBehaviorTracker.init('your-api-key', {
  enableAutomaticTracking: false
});

// Disable specific tracking types
const tracker = HumanBehaviorTracker.init('your-api-key', {
  automaticTrackingOptions: {
    trackButtons: false,  // Don't track buttons
    trackLinks: true,     // Still track links
    trackForms: true      // Still track forms
  }
});
```

## Troubleshooting

### Module Not Found

If you encounter module resolution issues:

```bash
# Clear cache and reinstall
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### Version Compatibility

Make sure you're using a compatible version:

```bash
# Check current version
npm list humanbehavior-js

# Update to latest version
npm update humanbehavior-js
```

### Ad Blocker Issues

If you're experiencing issues with ad blockers:

```javascript
// Test connection
const status = await tracker.testConnection();
if (!status.success) {
  console.log('Connection blocked by ad blocker');
}

// Check connection status
const connectionInfo = tracker.getConnectionStatus();
if (connectionInfo.blocked) {
  console.log('Connection blocked by ad blocker');
}
```

### Session Persistence

If sessions aren't persisting across page reloads:

```javascript
// Check if localStorage is available
if (typeof localStorage === 'undefined') {
  console.error('localStorage not available - sessions won\'t persist');
}
```
